# Avances de las prácticas #

** UART Por Software(Terminada) **
Esta práctica ya está terminada y probada con diferentes velocidades para verificar la fidelidad del programa. Esta se implementó con un timer en el cual tomamos un canal como output compare para hacer la transmisión y otro canal para la recepción. Para detectar 
el startbit utilizamos una interrupción de falling edge sobre ese pin.

** GSM/GPRS(Terminada) **
Esta práctica ya está lista. Se creó una interfaz entre el módulo y el MCU; éste puede mandar mensajes, leer mensajes entrantes y contestar llamadas.

** GPS(Terminada) **
Esta práctica ya está lista. El MCU recibe las tramas, pero solo hace caso a las cuales comienzan $GPRMC. En cuanto se hace la detección comienza a procesar. Después de procesarlo, lo guarda en una estructura donde se encuentran todos los datos contenidos en la trama.

** WIFI(Terminada) **
Esta práctica ya está lista. El MCU crea una conexión mediante el puerto 80. Desde una página web le mandamos los parámetros. El MCU recibe el frame de HTTP y obtiene el ID del cliente que envió y ls parámetros que le paso. Para probarlo utilizamos los LEDS integrados en la tarjeta. Después de detectar cerramos la conexión que realizó el cliente.

** RTC(Terminada) **
Es posible cambiar la fecha del RTC en tiempo de compilación, leerla de forma segura, escribirla en una pantalla LCD por I2C y utilizar la memoria RAM interna para propósitos generales.

** IMU(Terminada) **
La IMU es capaz de medir ángulos tanto en el eje X como en el eje Y con un error del +-5° máximo.
  
** Potenciómetro Digital(Terminada)**
El potenciómetro digital es capaz de modificar el valor de la resistencia entre el wiper y la terminal A o B. Al ser un potenciómetro de 100k con resolución de 7 bits, es posible cambiar la resistencia con incrementos de 781 ohm.

** Tarjeta SD (No terminada) **
Tenemos avances de la práctica y ya estamos intentando meter información a la SD, sin embargo, no hemos logrado ingresar la información por el momento, el pinstack a utilizar para la implementación de SPI ya está seleccionado.

** CAN (No terminada) **
El avance que tenemos de esta práctica es hacer la transmisión del FRAME de CAN. La recepción aun no la hemos implementado. El problema que tenemos con esta práctica es el ruido de la transmisión. 

** Cámara (No terminada) **
De esta práctica solo hemos hecho hacer que se tome la foto, pero hay problemas al guardarse en el buffer que está de por medio. A veces se toma la foto u otras el módulo de DCMI del MCU manda una flag de error. Faltaría investigar más acerca de este módulo interno del MCU.

**ZigBee (No terminada) **
Tenemos 3 módulos SMD ZigBee, pero no podemos comenzar la práctica porque necesitamos montarlos sobre una placa diseñada para poder sacarle los pines. Se tiene un SET de comandos AT para que, al momento de tener los módulos listos, formen una red estrella con un coordinador y dos dispositivos finales.
